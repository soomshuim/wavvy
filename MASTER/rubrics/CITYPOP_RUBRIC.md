# City Pop Genre Rubric

Version: 1.0
Last Updated: 2026-02-16
Purpose: 시티팝 장르 적합성 평가 루브릭
Scope: 장르가 City Pop인 트랙의 Seed Design / Variation Design QA
Pass Criteria: 80점 이상
Fail Criteria: 80점 미만 (재디자인 필수)

---

## 핵심 철학

> **시티팝은 빠르거나 느린 음악이 아니다. 도시를 '쿨하게' 기억하는 방식이다.**
>
> 시티팝은 장르라기보다 **'도시를 대하는 태도와 문법의 묶음'**이다.
> 아래 6개 팩터 중 다수가 충족되면 시티팝이다.

---

## 루브릭 (6개 팩터, 100점)

### 1. 도시 서사 (Urban Lyricism) — 20점

> 배경이 아니라 주인공이 '도시'다.

| 점수 | 기준 |
|------|------|
| 18-20 | 도시 오브젝트 4종+, "도시" 직접 명시, 도시 소리/냄새/빛 다중 감각, "세련된 외로움" |
| 14-17 | 도시 오브젝트 2-3종, 도시임을 알 수 있음, 감각 묘사 있음 |
| 10-13 | 도시 요소 1-2종, 도시/자연 구분 애매 |
| 5-9 | 도시 오브젝트 부재, 자연 이미지 중심 |
| 0-4 | 도시 요소 전무, 시골/자연과 구분 불가 |

**도시 오브젝트 예시**: 네온사인, 가로등, 교각, 택시, 아파트, 강변도로, 칵테일, 전화기, 드라이브, 골목, 간판

**핵심 원칙**:
- "사랑 노래 + 도시 배경" ❌ → "도시 안에서 발생한 감정" ⭕
- 사람의 감정보다 **공간의 정서**가 먼저 느껴져야 함
- 가사만 읽고 **도시임을 증명할 수 있어야** 함 (시골 강과 구분 불가 = 탈락)

---

### 2. 감정 절제 (Cool Restraint) — 15점

> 시티팝은 울부짖지 않는다.

| 점수 | 기준 |
|------|------|
| 14-15 | 감정 과잉 제로, 관조적 톤, "그냥 남아 있다" 태도, 감정을 설명 않고 남겨둠 |
| 11-13 | 대체로 절제, 1-2곳 소망/선언 표현 있으나 허용 범위 |
| 7-10 | 절제와 과잉 혼재, 감정 직접 서술 있음 |
| 0-6 | 감정 폭발, 고음 클라이맥스, 울부짖음 |

**핵심 원칙**:
- 고음 클라이맥스 ❌
- 감정의 폭발 ❌
- "아프다"보다 **"그냥 남아 있다"**
- 미묘한 이동, 체온 낮은 그리움
- 감정은 설명하지 않고, **남겨둔다**

---

### 3. 텐션 화성 (Tension Harmony) — 20점

> 시티팝은 멜로디보다 화성이 감정을 끌고 간다. 거의 필수 조건.

| 점수 | 기준 |
|------|------|
| 18-20 | Maj7/m7/9th/sus/slash + IV→iv 시그니처 + 확장 코드 명시 + "붕 뜬 몽환감" |
| 14-17 | 확장 코드 3종+ 명시, IV→iv 또는 Secondary Dom 있음 |
| 10-13 | 확장 코드 1-2종, 시그니처 전이 없음 |
| 5-9 | 단순 I-V-vi-IV 위주 팝 코드 |
| 0-4 | 화성 지정 없음 또는 3코드 단순 진행 |

**핵심 원칙**:
- Maj7, m7, m9, 13, sus, slash chord 적극 사용
- **IV→iv** 같은 감정 전이 = 시티팝 시그니처
- 단순 I-V-vi-IV 위주 팝 발라드 = **탈락**
- 뭔가 **붕 떠 있는 듯한** 세련되고 몽환적인 도시의 밤 분위기

---

### 4. 리듬 & 베이스 (Funky Groove) — 15점

> 멜로디는 부드럽지만 아래에서는 탄탄한 그루브.

| 점수 | 기준 |
|------|------|
| 14-15 | 슬랩 or 펑키 핑거 베이스, 16비트 그루브, 리듬 탑승감, BPM 적합 |
| 11-13 | 펑키 베이스 있음, 그루브 있으나 BPM이 경계선이거나 부분 적용 |
| 7-10 | 베이스 있으나 펑키하지 않음 (워킹/플랫), 그루브 약함 |
| 0-6 | 베이스 존재감 없음, 리듬감 부재 |

**핵심 원칙**:
- 슬랩 베이스 = 시티팝 정통 그루브 (100+ BPM에서 최적)
- 느린 템포(~90 BPM)에서는 **mellow round finger bass**가 적합
- 16비트 하이햇 그루브 = 시티팝 리듬 필수
- 리스너가 **리듬을 타게** 만들어야 함

**BPM별 베이스 가이드**:
| BPM | 권장 베이스 |
|-----|-----------|
| 100+ | Slap bass, punchy groove |
| 90-100 | Funky finger bass, subtle slap |
| 80-90 | Mellow round finger bass (슬랩 주의: 펑크 발라드 위험) |

---

### 5. 사운드 팔레트 (Vintage Sound) — 20점

> 80년대 일본 팝이 만들었던 '도시의 소리'를 이해했는가.

| 점수 | 기준 |
|------|------|
| 18-20 | 7개 요소 중 6-7개 충족 |
| 14-17 | 7개 요소 중 4-5개 충족 |
| 10-13 | 7개 요소 중 2-3개 충족 |
| 5-9 | 7개 요소 중 1개 충족 |
| 0-4 | 빈티지 사운드 요소 전무 |

**7개 필수 요소**:

| # | 요소 | 설명 | Style 키워드 예시 |
|---|------|------|-----------------|
| 1 | **Juno 계열 아날로그 신스** | 맑고 따뜻한 패드 | `Juno-106 analog synth pad` |
| 2 | **DX7 FM 신스** | 80s EP 사운드 | `DX7 FM electric piano` |
| 3 | **Brass** | 화려한 금관 섹션/스탭 | `brass stabs`, `soft brass` |
| 4 | **코러스 클린 기타** | JC-120 계열 | `clean guitar with chorus effect` |
| 5 | **Plate Reverb** | 80s 공간감 | `lush plate reverb`, `spacious floating` |
| 6 | **Tape warmth** | 아날로그 질감 | `tape warmth`, `tape saturation` |
| 7 | **라이브 느낌 드럼** | 브러시/오픈 하이햇 | `brush kit`, `live drums` |

**핵심 원칙**:
- "빈티지 흉내" ❌ → **"80년대 일본 팝의 도시 소리를 이해했는가"** ⭕
- 시티팝은 리버브와 코러스를 풍성하게 → **Dry 주의** (스튜디오 인디밴드 위험)
- 소리가 약간 **공중에 떠 있어야** 제맛
- `sparkling`, `mellow` = 시티팝 광택 키워드

**Dry vs Spacious 주의사항**:
```
❌ Raw, Solid, Direct, Dry → 시티팝 공간감 파괴, 인디밴드 느낌
⭕ Raw, Solid, Direct + Lush plate reverb → 진성 보컬 + 시티팝 공간감 공존
```

### 6. 장르 정체성 (Genre Identity) — 10점

> 시티팝은 애매하면 안 된다.

| 점수 | 기준 |
|------|------|
| 9-10 | "시티팝이다" 즉시 인식, 사운드+화성+정서 삼중 방어 |
| 7-8 | "시티팝이다" 방어 가능, 1개 요소 약함 |
| 4-6 | "시티팝인가? R&B인가?" 애매 |
| 0-3 | 시티팝으로 분류 불가 |

**핵심 원칙**:
- R&B인가? 발라드인가? 재즈인가? → ❌
- **"아, 시티팝 계보다"**라는 첫인상이 있어야 함
- 최소한 톤·화성·정서 중 **하나는 강하게 각인**되어야 함

## 체크리스트 (간이 판정)

> 아래 중 3개 이상이면 시티팝, 4개면 확실, 5개면 정통.

```
□ 도시가 감정의 주체다
□ 감정이 절제되어 있다
□ 재즈/소울 화성 문법을 쓴다
□ 80s 일본 팝의 사운드 계보가 들린다
□ "이건 시티팝이다"라고 방어 가능하다
```

## QA 워크플로우

```
Step 1. Seed/Variation 디자인 완료 (가사 + Style + Exclude)
Step 2. 기존 QC 통과 확인 (가사 체크리스트 + S0-S20 + 글자수)
Step 3. 장르가 City Pop인 경우 → City Pop Rubric 실행
Step 4. 6개 팩터별 채점 + 총점 산출
Step 5. 판정:
        ≥ 80점 → PASS (제안 가능)
        < 80점 → FAIL (재디자인 필수, 유저에게 제안 금지)
        개별 팩터 5점 이하 → 해당 팩터 CRITICAL FAIL (총점 무관 재디자인)
Step 6. PASS 시 채점표와 함께 output
```

**CRITICAL FAIL 규칙**: 총점이 80 이상이어도 **개별 팩터 5점 이하**가 하나라도 있으면 FAIL.
